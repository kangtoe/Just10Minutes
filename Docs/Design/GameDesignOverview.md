# AutoSpaceShooter - 게임 디자인 전반

## 프로젝트 개요

로그라이크 방식의 2D 스페이스 슈팅 게임. 기존 제작 중이던 슈팅 게임의 일부 스크립트를 계승하여 새롭게 제작.

## 핵심 컨셉

**"간단한 조작, 전략적인 선택"**
Asteroids 스타일의 회전 메커닉과 Vampire Survivors 스타일의 업그레이드 시스템을 결합한 생존형 게임입니다.
플레이어는 자동으로 전진하는 우주선을 **회전만으로 조종**하며, 적을 최대한 회피, 혹은 격파하며 오래 생존하는 것이 목표입니다.
기존 게임의 복잡하고 난잡한 조작 방식을 개선하여, 단순한 조작으로 플레이 가능하면서도 전략적 선택을 통한 재미를 극대화.

## 핵심 게임플레이

### 2.2 게임 플레이 루프

```
시작
  ↓
자동 전진 + 회전 제어로 이동
  ↓
적 충돌 회피 (적과 충돌 시 피해)
  ↓
적 격파 시 경험치 획득
  ↓
레벨업 시 업그레이드 선택
  ↓
더 강한 적 등장
  ↓
(사망 시) 점수 기록 및 재시작
```

### 조작 방식
- **자동 전진**: 플레이어 기체는 바라보는 방향으로 자동 이동
- **자동 사격**: 플레이어 기체가 자동으로 전방 사격
- **회전 조작**: 화면 좌/우 절반을 클릭/터치하여 기체 회전 방향 조절
- **원터치 조작**: 회전만 제어하면 되므로 간단한 터치 조작으로 플레이 가능

### 게임 규칙

#### 즉사 조건
- **사망 조건 1**: 체력이 0이 됨
- **사망 조건 2**: 화면 경계를 벗어남 (즉사)

#### 전투 시스템
- **적/장애물**: 대부분 접촉 시 즉사가 아닌 일정량 피해만 가함
- **공격 방식**:
  - **자동 사격**: 플레이어 기체가 자동으로 사격 (수동 조작 불필요)
  - **충돌 공격**: 적과의 직접 충돌로 피해 가함
- 두 가지 공격 방식을 활용한 다양한 빌드 가능
  - 사격 중심 빌드 (연사, 관통, 멀티샷 등)
  - 충돌 중심 빌드 (충돌 피해 강화)

#### 물리 기반 이동
- Rigidbody2D.AddForce() 사용
- transform.up 방향으로 지속적인 힘 가함
- 적과의 충돌 시 물리 반응 발생

## 게임 특징

### 로그라이크 시스템 (노바 드리프트 참고)

#### 기본 진행 루프 (Phase 1)
1. **경험치 획득**: 적 격파 시 경험치 획득
2. **레벨업**: 경험치가 일정량 누적되면 레벨업
3. **업그레이드 선택**: 레벨업 시 여러 업그레이드 중 하나 선택
   - 체력, 연사속도, 멀티샷, 이동속도, 회전속도, 발사체 피해/속도, 충돌 피해 등
   - 상세 스탯 정의: [PlayerStats.md](PlayerStats.md) 참조
4. **밸런싱**: 의도한 시간마다 한 번씩 레벨업이 발생하도록 경험치량 조절

#### 확장 요소 (Phase 2-3)
- **장비 카테고리 시스템**: 무기/실드/바디 등 카테고리별 장비
- **리롤 시스템**: 업그레이드 선택지 재선택 가능
- **메타 진행**: 런 반복 시 영구 언락 요소
- **시너지 시스템**: 업그레이드 조합으로 강력한 빌드 생성

### 빌드 다양성
- 사격 중심 빌드
- 충돌 중심 빌드
- 기타 빌드 (추후 확장)

## 이전 프로젝트와의 차이점

| 항목 | 이전 게임 | 현재 게임 |
|------|----------|----------|
| 조작 방식 | 수동 조작 (복잡) | 자동 이동 + 회전만 조작 |
| 조작 난이도 | 높음 | 낮음 |
| 재미 요소 | 조작 숙련도 | 전략적 선택 |
| 스크립트 | 처음부터 작성 | 일부 재사용 |
