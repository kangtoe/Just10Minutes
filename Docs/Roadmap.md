## Phase 2: 전투 및 성장 시스템 깊이 확대 (현재 진행 중)

### 성장 시스템 확장 (최우선)
- [ ] **업그레이드 옵션 대폭 확대** (현재 기본 스탯 위주 → 10-20개로)
  - 현재: 체력, 연사, 이동속도, 회전속도 등 기본 스탯
  - 추가 필요:
    - [ ] 관통 (Penetration) - 발사체가 적을 관통
    - [ ] 멀티샷 (Multi Shot) - 여러 발사체 동시 발사
    - [ ] 충돌 피해 증가 (Collision Damage)
    - [ ] 실드 재생 (Shield Regeneration)
    - [ ] 경험치 획득량 증가
    - [ ] 발사체 속도
    - [ ] 발사체 크기
    - [ ] 넉백 강도
    - [ ] 치명타 확률/피해
    - [ ] 드론 시스템 (Phase 3에서 이동 고려)
- [ ] **빌드 시너지 시스템 설계 및 구현**
  - [ ] 사격 빌드 (High DPS)
    - 멀티샷 + 연사 + 발사체 피해
    - 약점: 근접 위험, 실드 부족
  - [ ] 충돌 빌드 (Tank/Ramming)
    - 충돌 피해 + 최대 체력 + 넉백
    - 약점: 원거리 적에 약함
  - [ ] 기동 빌드 (Dodge/Kiting)
    - 이동속도 + 회전속도 + 실드 재생
    - 약점: 화력 부족
  - [ ] 하이브리드 빌드
    - 각 빌드 조합 시너지
- [ ] 업그레이드 희귀도 시스템 (Common/Rare/Epic)

## 기술 부채 및 개선 사항

### 아키텍처 개선
- [ ] **싱글톤 의존성 완화**
  - 현재: 모든 매니저가 MonoSingleton
  - 문제: 단위 테스트 어려움, 결합도 높음
  - 개선안: 의존성 주입 또는 ScriptableObject 기반 아키텍처 고려
- [ ] **레이어 및 태그 중앙 관리**
  - 현재: 하드코딩 (레이어 7, 레이어 128 등)
  - 개선안: LayerMask, Tag를 const 또는 enum으로 중앙화
- [ ] **매니저 통합 검토**
